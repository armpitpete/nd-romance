# ND Readable Romance Workflow and Use Order v1.1

## Core use

This file locks the **working order** for the pack.

Its job is to stop:

- random drafting
- structure being skipped
- scene work happening before the relationship engine is clear
- diagnosis happening too late
- revision happening in the wrong order
- stale or superseded files being treated as live

---

## Core rule

Use the system in workflow order.

Normal order:

1. define the romance engine
2. map the structure
3. map progression across the book
4. set up scene and beat control
5. draft with active checking
6. diagnose failure precisely
7. revise in the right sequence
8. only then do proof and polish

# Working order by phase

## Phase 1 — define the book before drafting

Open, in order:

1. `ND_Readable_Romance_Design_Sheet_v1.0.md`
2. `ND_Readable_Romance_Structure_Template_v1.0.md`
3. `ND_Readable_Romance_Arc_Map_v1.0.md`

### Exit condition

Do not begin drafting until these are clear:

- why these two
- what the central fault line is
- how the relationship changes across the book
- what rupture and repair will require
- what the ending must prove

---

## Phase 2 — set up drafting control

Open, in order:

1. `ND_Readable_Romance_Scene_Card_Template_v1.0.md`
2. `ND_Readable_Romance_Scene_Card_v1.0.md`
3. `ND_Readable_Romance_Beat_Tracker_v1.0.md`

### Exit condition

Do not continue into free drafting unless:

- scenes have a readable job
- chapter movement can be tracked
- the middle is not just a pile of interactions

---

## Phase 3 — draft with active checking

Use repeatedly:

- `ND_Readable_Romance_Scene_Card_v1.0.md`
- `ND_Readable_Romance_Beat_Tracker_v1.0.md`
- `ND_Readable_Romance_Checklist_v1.0.md`

### Purpose

Keep the manuscript legible while it is still being made.

### Ask during drafting

- what changed in this scene
- what changed in this chapter
- whether the bond deepened, strained, or clarified
- whether the scene has one primary function
- whether the manuscript is repeating instead of progressing

---

## Phase 4 — diagnose before rewriting

Open:

- `ND_Readable_Romance_Draft_Failure_Triage_Sheet_v1.0.md`

Use this when the book feels:

- flat
- false
- repetitive
- underpowered
- emotionally vague
- broken at the rupture
- weak at the ending

### Exit condition

You must be able to name:

- the dominant failure
- where it starts
- whether it is local or structural

---

## Phase 5 — revise in order

Open:

- `ND_Readable_Romance_Revision_Workflow_v1.1.md`

### Repair order

1. engine
2. structure
3. progression
4. scene movement
5. readability and ND truth
6. rupture and repair proof
7. line-level polish

---

## Phase 6 — control check and closure

Open:

- `ND_Readable_Romance_Master_Index_v1.1.md`
- `ND_Readable_Romance_Pack_Review_and_Control_Note_v1.1.md`
- `ND_Readable_Romance_Overlap_File_Resolution_Note_v1.1.md`

### Purpose

Use these files to confirm that:

- the pack is internally consistent
- no stale filename is still being treated as live
- the control layer matches the actual contents of the pack

# Rescue routing

## If the premise is promising but fuzzy

Go back to:

- `ND_Readable_Romance_Design_Sheet_v1.0.md`

## If the middle drags

Go to:

- `ND_Readable_Romance_Beat_Tracker_v1.0.md`
- `ND_Readable_Romance_Arc_Map_v1.0.md`

## If the couple do not feel specific

Go to:

- `ND_Readable_Romance_Design_Sheet_v1.0.md`
- `ND_Readable_Romance_Checklist_v1.0.md`

## If the breakup feels fake

Go to:

- `ND_Readable_Romance_Structure_Template_v1.0.md`
- `ND_Readable_Romance_Arc_Map_v1.0.md`
- `ND_Readable_Romance_Draft_Failure_Triage_Sheet_v1.0.md`

## If the ending lands weakly

Go to:

- `ND_Readable_Romance_Arc_Map_v1.0.md`
- `ND_Readable_Romance_Checklist_v1.0.md`

---

# What not to do

Do not:

- jump into prose because the chemistry idea feels strong
- skip structure because the characters feel vivid
- use the checklist as a substitute for the triage sheet
- revise local scenes before confirming the book-level failure
- keep both the old `ND_Romance_*` revision file and the corrected canonical replacement active
