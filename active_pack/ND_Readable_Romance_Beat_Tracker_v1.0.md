# ND Readable Romance Beat Tracker v1.0

## Purpose

Use this tracker while drafting or revising a live manuscript.

It is a **working grid** for checking whether each romance beat is:

- clear
- necessary
- emotionally legible
- doing actual relationship work

This is not the full theory file.  
This is the **quick control sheet** you keep beside the manuscript.

---

# One-line lock

**Every beat must either reveal the bond, strain the bond, clarify the bond, or change the bond.**

---

# Use rule

For each scene or chapter beat, fill in **one row only**.

If you cannot fill the row clearly, the beat is probably:

- vague
- repetitive
- under-built
- or not yet doing enough

# Allowed beat jobs

Use short labels only.

- first notice
- first ease
- attraction sharpened
- trust opens
- care shown
- pattern repeated
- masking shown
- unmasking shown
- misread begins
- misread worsens
- misread clarified
- strain introduced
- vulnerability offered
- vulnerability refused
- rupture
- repair attempt
- repair achieved
- desire spoken
- retreat
- choice forced
- incompatibility exposed
- commitment test
- ending proof

# Column guide

## Beat job
What this beat is for.

If the beat has no clear job, it is probably not ready.

## Start relationship state
Use short phrases like:

- strangers
- wary
- curious
- fragile ease
- active attraction
- unstable trust
- opened
- ruptured
- repairing
- committed but strained

## What A thinks is happening / What B thinks is happening
Do not leave this vague.

This is one of the main ND readability controls.

## What is actually happening
Write the emotional truth of the beat in one sentence.

## Bond / attraction evidence
Name the **specific thing** that makes the bond legible here.

Examples:

- notices overload early
- remembers exact preference
- returns after avoidance
- makes space without demanding speech
- sees through performance
- chooses precision over charm
- offers care without pressure

## Pressure / misread / regulation factor
Track what is complicating the beat.

Examples:

- noise overload
- public setting
- shame response
- delayed processing
- masking
- jealousy
- fear of burdening
- care misread as control
- silence misread as rejection

## End change
Complete this sentence:

**By the end of this beat, now...**

If you cannot do that, the beat is weak.

## Keep / cut / revise
Use only:

- keep
- cut
- revise

---

# Quick pass rules

## Keep
Use **keep** if all are true:

- [ ] the beat has a clear job
- [ ] the bond or strain becomes more legible
- [ ] the emotional logic is trackable
- [ ] something changes

## Cut
Use **cut** if any are true:

- [ ] the beat repeats an earlier beat without adding pressure or clarity
- [ ] the scene runs on vibe only
- [ ] no real relationship change occurs
- [ ] the beat exists only to fill space between larger moments

## Revise
Use **revise** if any are true:

- [ ] the beat job is valid but muddy
- [ ] the bond is asserted but not shown
- [ ] overload, silence, avoidance, and rejection are blurred
- [ ] the misread is too vague
- [ ] the end change is under-earned

---

# ND readability stress test

After filling the grid, check:

- [ ] I can follow the romance row by row.
- [ ] I can see where trust changes.
- [ ] I can see where misreads begin and end.
- [ ] I can tell why the pair matters.
- [ ] I can distinguish care from pressure.
- [ ] I can distinguish overload from rejection.
- [ ] The relationship does not stall in repeated almost-scenes.
- [ ] The ending rows are earned by the earlier rows.

---

# Red-flag patterns in the grid

Revise if the tracker shows any of these:

- [ ] many rows have the same beat job
- [ ] “what is actually happening” stays fuzzy for multiple rows
- [ ] bond evidence is generic in most rows
- [ ] pressure comes only from vagueness
- [ ] end changes are tiny for too long
- [ ] repair is missing after major rupture
- [ ] attraction appears stronger than trust without explanation
- [ ] the final commitment arrives before the grid has earned it

---

# Plain-language proof

The tracker is working if I can read down the rows and explain:

- how the bond formed
- how the strain developed
- where each person misread the other
- what changed the relationship
- why the ending is deserved

If not, the manuscript still has fog in it.

---

# Final lock

**A readable romance draft should look like a sequence of clear relational changes, not a pile of moods.**

Depth is welcome.  
Fog is not.
